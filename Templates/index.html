Make An Sqlite3 DataBase

Super Maintainable, Faster and Easily

The lite3 module is a set of classes and functions fully defined to making the use of the sqlite3 Python
builtin database module easy, improving code understanding without unfamiliar MySQL syntax.

Compare the difference in speed and easiness of lite3 and vanilla sqlite3 python from the two videos.

Creating a database and connection

This is similar to the normal where way of creating the database and connecting to it in Sqlite3 but just because we assign a variable to handle the class and its parameters since you are to use it throughout your code.

The parameters include;


  1. Database_name
  2. Connect You can use any string word
  3. Cursor Again you can use any string word
	  					
import sqlite3
import lite3

mydb = lite3.lite('customers.db', 'conn', 'cursor')

Remember the connection and cursor parameters are needed, you can use any string word.  Likely, you won't use them again.
The cursor is also made at this point with the connection and the database.

Creating a Table and it's Columns

We use the execute(  ) function from the lite class to carryout this task. It will take two parameters

The parameters include;

  1. Table
  2. column this must be a list

The column parameter defines the number of columns to be used in you table.
It is presented in a list .

	  					
table_columns = ['name', 'email']
mydb.execute( 'workers', table_columns )

This will create a table named workers with 2 columns (  name,  email  ).
You can add as many as you want.

Inserting or Adding Values/Rows in the Table

Use the insert(   ) function to do this but with two parameters

Parameters include;

  1. Table the name of the table you want ot use.
  2. rows this can be a list with many or one tuple or a single tuple

The tuple or tuples must have all values according to the specified columns.
Any missing will raise an error.
You must call the execute(  ) function first before the insert(  ) function.

	  					
mydb.execute('workers', table_columns)

values_1 = ('jack', 'jhav@web.com')
values_2 = [('Rogers', 'rogers@gmail.com'), ('Caty', 'rowcat@gmail.com')]
# all values must be filled in the columns.

mydb.insert('workers, value_2)

This must be called once when needed or else it will create many values in the database.
You can use a single tuple with the row values or a list with tuples that have row values.

   You must call the execute(  ) function first before using the insert(  ) function since it uses some resources from it.

Fetching items

You can use very many function for this.

Fetching one

A simple function to do this is fetchone(  ) and it takes one parameter

parameter includes;

  1. Table
	  					
one = mydb.fetchone('workers')
print(one)
	  				
	  					
>>> ('Rogers', 'rogers@gmail.com')

As usual it will return the first row from the database table given.

Fetching many

For the fetchmany(  ) function, you will need two parameters

parameters include;

  1. Table
  2. Number this is the number of rows you want
	  					
many_one = mydb.fetchmany('workers', 3)
many_two = mydb.fetchmany('workers', 1)
print(many_one)
print(many_two)
	  				
	  					
>>> [('Rogers', 'rogers@gmail.com'), ('Caty', 'rowcat@gmail.com'), ('jack', 'jhav@web.com')]
>>> [('Rogers', 'rogers@gmail.com')]

A list from the database will be returned with the number of rows defined in the parameter.

Fetching all

A simple function to do this is fetchall(  ) and it takes one parameter

parameter includes;

  1. Table
	  					
all_rows = mydb.fetchall('workers')
print(all_rows)
	  				
	  					
>>> [('Rogers', 'rogers@gmail.com'), ('Caty', 'rowcat@gmail.com'), ('jack', 'jhav@web.com')]

Remember I have 3 rows in my database and a list of all of them has been returned.

Fetching With IDs or ROWID

The functions are the same but just starting with id like id_fetchall(  ), id_fetchmany(  )

Fetching one with a Rowid

The simple function for this is id_fetchone(  ) and it takes one parameter

parameter includes;

  1. Table
	  					
one = mydb.id_fetchone('workers')
print(one)
	  				
	  					
>>> ('1', 'Rogers', 'rogers@gmail.com')

As usual it will return the first row from the database table.

Fetching many with Rowids

For the id_fetchmany(  ) function, you will need two parameters

parameters include;

  1. Table
  2. Number this is the number of items you want
	  					
many_one = mydb.id_fetchmany('workers', 3)
many_two = mydb.id_fetchmany('workers', 1)
print(many_one)
print(many_two)

[('1', 'Rogers', 'rogers@gmail.com')]

A list from the database will be returned with the number of rows defined in the parameter.

Fetching all with rowid

A simple function to do this is id_fetchall(  ) and it takes one parameter

parameter includes;

  1. Table
	  					
all_rows = mydb.id_fetchall('workers')
print(all_rows)
	  				
	  					
[('1', 'Rogers', 'rogers@gmail.com'), ('2', 'Caty', 'rowcat@gmail.com'), ('3', 'jack', 'jhav@web.com')]

Remember I have 3 rows in my database Tbale workers and a list of all of them is returned.

Finding or Querying Database data.

We can use very many ways to do this,  and again I have provided very many functions to do this but it is easy to use them.
Since they are many, choose the one that can do your task.
Just follow the code below.

Using like/LIKE

To find great use of LIKE I have provided three functions

  • find_like_start You can use this one to find values like starting with specified letters or a word or anything  find_like_start()
  • find_like_any this one is used to find specified values either starting ending or in the middle .  find_like_any( )
  • find_like_end this one is used to find specified values at the end . find_like_end( )

All these require the use of three parameters.

  • Table
  • Column this is where to choose from the data to specify.
  • string this is the word, letter or letters you want to use to search or query the data in the database to be returned.

Find like at the start

Also this will require three parameters just like the others.

  • Table
  • Column Column from which you want to query.
  • String Word to use to query the rows or values.
	  					
quer = mydb.find_like_start( 'workers', 'name', 'Ro' )
print(quer)

quer2 = mydb.find_like_start( 'workers', 'email', 'ro' )
print(quer2)
	  				
	  					
>>> [(Rogers', 'rogers@gmail.com')]

>>> [('Rogers', 'rogers@gmail.com'), (Caty', 'rowcat@gmail.com')]

They are all presented in a list although it is one.

Find like anywhere

Also this will require three parameters just like the others.
And remember it will choose either from the inside, end or start.

  • Table
  • Column Column from which you want to query.
  • String Word to use to query the rows or values.
	  					
quer3 = mydb.find_like_any('workers', 'name', 'a')
print(quer3)

quer4 = mydb.find_like_any('workers', 'email', '.com')
print(quer4)
	  				
	  					
>>> [('jack', 'jhav@web.com'), (Caty', 'rowcat@gmail.com')]

>>> [('Rogers', 'rogers@gmail.com'), ('Caty', 'rowcat@gmail.com'), ('jack', 'jhav@web.com')]

In the names, one does not have letter a anywhere so it missed but for the emails, all of them have .com , so they were all presented back.

Find like at the end

Also this will require three parameters just like the others.
Remember the last letter specified must be the last letter of the value you want.

  • Table
  • Column Column from which you want to query.
  • String Word to use to query the data.
	  					
quer5 = mydb.find_like_end('workers', 'name', 'ck')
print(quer5)

quer6 = mydb.find_like_end('workers', 'email', 'om')
print(quer6)
	  				
	  					
>>> [('jack', 'jhav@web.com')]

>>> [('Rogers', 'rogers@gmail.com'), ('Caty', 'rowcat@gmail.com'), ('jack', 'jhav@web.com')]

In the names, only Jack ends with ck but all emails end with om so they were all presented back accordingly.

Using Operators and MySQL syntax

This is used to find the exact data you want from the database.

I have one function for this find_no_query( ) but it will require two parameters

parameters include;

  • Table
  • string This string should have a specifying sentence or separated words.

      Using examples from our database

    • va = '  name = "Rogers"  '
      mydb.find_no_query('workes', 'va')

      >>> [('Rogers', 'rogers@gmail.com')]

      • Let me say we had age column that uses numbers

    • va2 = '  age < 18  '
      mydb.find_no_query('workes', 'va2')

      >>> [('Rogers', 'rogers@gmail.com', 16), ('Kelly', 'kelly@heruko.com', 17)]

      • Remember that in the string if you are using one of the database values or word but not a column name,
      add other quotation marks either double or single but opposite to the parent ones and for numbers just leave the way they are.
      Follow python str and int syntax.

Finding exact data with find_no_query

Remember with this,   you can find the exact data you want.

	  					
string1 = ' name = "Rogers" '

quer7 = mydb.find_no_query('workers', string1)
print(quer7)

string2 = ' email = "jhav@web.com" '

quer8 = mydb.find_no_query('workers', string2)
print(quer8)
	  				
	  					
>>> [('Rogers', 'rogers@gmail.com'),]

>>> [('jack', 'jhav@web.com')]

Remember to add other quotations for the word you want to find in the string parameter of the find_no_query( ) function.

Updating Database Data.

Since we are updating, we shall need to specify where and what to do to that point.
We have to first specify what should be done and then where it should be done.
This is done through using parameters.

parameters include;

  1. Table
  2. Column where the update should take placeThis is the column to change.
  3. New value for the specified valueThis is the new data to be set to that place.
  4. Which column to find This is the column to find the value to be changed.
  5. value This is the value to changed

Normally

	  					
print(mydb.id_fetchall('workers'))
mydb.update('workers', 'name', 'Frank', 'name', 'jack')
# jack is going to be changed to Frank

view_change = mydb.fetchall('workers')
print(view_change)
	  				
	  					
>>> [('Rogers', 'rogers@gmail.com'), ('Caty', 'rowcat@gmail.com'), ('jack', 'jhav@web.com')]
>>> [('Rogers', 'rogers@gmail.com'), ('Caty', 'rowcat@gmail.com'), ('Frank', 'jhav@web.com')]

With Rowid

For the use of rowid, you have to set parameter 4 as 'rowid',
this will tell that you want to use rowid and remember the rowid is a number so use a number for the last parameter.

	  					
print(mydb.id_fetchall('workers'))

mydb.update('workers', 'name', 'Kabuye', 'rowid', 1)
# Rogers is going to be changed to Kabuye

print(mydb.id_fetchall('workers'))
	  				
	  					
>>> [('1', 'Rogers', 'rogers@gmail.com'), ('2', 'Caty', 'rowcat@gmail.com'), ('3', 'Frank', 'jhav@web.com')]

>>> [('1', 'Kabuye', 'rogers@gmail.com'), ('2', 'Caty', 'rowcat@gmail.com'), ('3', 'Frank', 'jhav@web.com')]

Deleting Database Data.

This could be similar to updating but here it requires two parameters and one is special

  1. Table
  2. Datatype This should be a tuple or int/rowid or str.

    Using a tuple for datatype parameter.


    The tuple must have three items with descriptive information like;
    tuple_1 = ('column', 'operator', 'row_value')
    tuple_2 = ( 'name', '=', 'Kabuye' )
    tuple_3 = ( 'age', '<', 15 )
    tuple_4 = ( 'email' = 'rowcat@gmail.com')' ) 

	  					
viewer1 = mydb.fetchall('workers')
print(viewer1)

tuple_5 = ( 'name', '=', 'Caty')
mydb.delete('workers', tuple_5)
#the item is deleted for good, no undo.

print(viewer1)
	  				
	  					
>>> [('1', 'Kabuye', 'rogers@gmail.com'), ('2', 'Caty', 'rowcat@gmail.com'), ('3', 'Frank', 'jhav@web.com')]

>>> [('1', 'Kabuye', 'rogers@gmail.com'), ('3', 'Frank', 'jhav@web.com')]

Using str or string

A string with a descriptive sentence;
string_1 = ' row = "column" '
string_2 = 'name = "Kabuye" '
string_3 = ' age < 15 '
string_4 = ' email = "rowcat@gmail.com" '

mind the quotation marks where it is a string or value and not a column

Limiting Data to Fetch From the Database

Limit Fetching all

To limit the rows or data you want to fetch from the database, I have provided the limit function where it will have to present all the data but with a limit from one of the parameters of the limit( ) function.

parameters include;

  1. Table
  2. Limit this should be a number
	  					
limit_a = mydb.limit('workers', 2)
print(limit_a)

limit_b = mydb.limit('workers', 1)
print(limit_b)
	  				
	  					
>>> [('Rogers', 'rogers@gmail.com'), ('Caty', 'rowcat@gmail.com')]

>>> [('Rogers', 'rogers@gmail.com')]

Note: if the limit number exceeds the number of data or rows in the database,  it will return only all it has.

Limiting Fetching all with Rowid

All is the same but the id_limit(  ) function just brings back the data with its rowid and it also takes two parameters

parameters include;

  1. Table
  2. Limit this should be a number
	  					
limit_c = mydb.id_limit('workers', 2)
print(limit_c)

limit_d = mydb.id_limit('workers', 1)
print(limit_d)
	  				
	  					
>>> [(1, 'Rogers', 'rogers@gmail.com'), (2, 'Caty', 'rowcat@gmail.com')]

>>> [(1, 'Rogers', 'rogers@gmail.com')]

Note: if the limit number exceeds the number of data or rows in the database, it will return only all it has.

Ordering Database Data with or without a limit

This is a one function that can do it all but it requires parameters according to what you want, if you want a limit in your results or fetch, you need to provide different parameters or you want to use the rowid again different parameters.

'A'   for ascending
'D'   for descending

  • Easy way

      In this way we have to use two parameters

    • Table
    • Order either ascending or descending.
      use 'A' to represent acsending,
      'D' to represent descending

      • Using acsending order


    • order_1 = mydb.order('workes', 'A')
      print(order_1)

      >>> [('1', 'Kabuye', 'rogers@gmail.com'), ('2', 'Caty', 'rowcat@gmail.com'), ('3', 'Frank', 'jhav@web.com')]

      • Using acsending order with a limitJust add the number of rpws or data you want as the third parameter.


    • order_2 = mydb.order('workes', 'A', 2)
      print(order_2)

      >>> [('1', 'Kabuye', 'rogers@gmail.com'), ('2', 'Caty', 'rowcat@gmail.com')]

      • Using decsending order


    • order_1 = mydb.order('workes', 'B')
      print(order_1)

      >>> [('3', 'Frank', 'jhav@web.com'), ('2', 'Caty', 'rowcat@gmail.com'), ('1', 'Kabuye', 'rogers@gmail.com'),]

      • Using decsending with a limitJust add the number of rows or data you want as the third parameter.


    • order_2 = mydb.order('workes', 'B', 2)
      print(order_2)

      >>> [('3', 'Frank', 'jhav@web.com'), ('2', 'Caty', 'rowcat@gmail.com')]
  • Order with description

    For this one, we use the a list with two items for the second parameter of the order( ) function.

    • Table
    • Order A list with two items is required.
      • list1 = [ 'name',  'A' ]
        This means we are going to use the name's first letters to decide who comes first.
      • list2 = [ 'email',  'D' ]
        This means we are going to use the emails's first letters to decide who comes last.


      • #The order comes uses the column name"s Alphabetical order
        list1 = [ 'name', 'A']

        order_3 = mydb.order('workes', list1)
        print(order_3)

        >>> [('2', 'Caty', 'rowcat@gmail.com'), ('3', 'Frank', 'jhav@web.com'), ('1', 'Kabuye', 'rogers@gmail.com')]


      • #The order comes uses the column email"s Alphabetical order and I have added a limit
        list2 = [ 'email', 'D']
        list3 = [ 'age', 'D']

        order_4 = mydb.order('workes', list2, 2)
        print(order_4)

        >>> [('2', 'Caty', 'rowcat@gmail.com'), ('1', 'Kabuye', 'rogers@gmail.com')]

For numbers, it will go as usual 'A' small to big and the rest verse-verser.
You can Add a number as the last parameter for your limit.
